## 3. 計算ロジック（クライアント仕様 2026-05-15 反映済み）

### 3-1. フリー在庫（クライアント最終仕様 2026-05-15・簡素化版）

引当済と予約未処理は同義のため**予約未処理**を採用。0.7逆算は不要。
**通常タイプ・予約タイプ共通**で次の1式：

```
フリー在庫 = 在庫 + 入荷残 − 予約未処理
```

- 在庫       = 倉庫在庫（inventory_snapshot）
- 入荷残     = 既発注未入荷（incoming_stock, MMS/Tableau）
- 予約未処理 = 予約受注がついている分（予約管理一覧の未処理数 quantity）
- 通常タイプは予約未処理が無いので実質「在庫 + 入荷残」になる。
- 旧仕様（予約受付数 ÷ 0.7・販売前予約受付数）は廃止。
- ⚠️ **依存**: 入荷残（MMS/Tableau）がまだ未連携。連携完了まで入荷残=0で算出されます。

### 3-2. 販売速度（日販）— 既存品番の新色対応

シタテル確定リリース日は【品番ごと】で同一品番の新色を区別できないため、
**SKU（=カラー×サイズ）ごとの実初回受注日**を有効販売開始日とする。
受注データ上の実際の初回注文日なのでZOZOの不正確な日付項目とは別物。

```
有効販売開始日 = そのSKU(カラー)の実初回受注日（受注データの最古日）
30日販売速度 = 直近30日販売数 ÷ MIN(30, @target_date − 有効販売開始日 + 1)
 7日販売速度 = 直近 7日販売数 ÷ MIN( 7, @target_date − 有効販売開始日 + 1)
```

→ 同一品番でも「ブラック(既存30日実績)は÷30」「ベージュ(新色7日実績)は÷7」と
カラー別に公平評価。再販スパイクの旧品番も初回受注日が古いので新商品誤判定なし。

### 3-3. 推奨発注数

```
推奨発注数 = MAX(0, CEIL(8週 × 7日 × 30日販売速度 − フリー在庫))
```

- 入荷残はフリー在庫に含むため、ここで重ねて引かない。
- ⚠️ **要確認**: 「8週」はベース実装。Manus本来の「ピーク月選択方式 ＋ 52週季節係数」への
  切替も可能（Phase 2）。8週固定か Manus方式か要ご相談。

### 3-4. 粗利率

```
売上(30日) = 受注CSV「合計金額（税抜）」の30日合計
原価(30日) = MMS原価 × 受注数の30日合計
粗利率     = (売上 − 原価) ÷ 売上 × 100
```

### 3-5. 緊急度・在庫日数

```
在庫日数   = 真のフリー在庫 ÷ 7日販売速度
緊急度: CRITICAL = フリー在庫 ≤ 0
        WARNING  = 在庫日数 ≤ 14日
        OVERSTOCK= 在庫日数 > 90日
        OK       = それ以外
欠品まで日数 = 在庫日数（整数）
```
