## 📌 設計書 = What / How / Where の定義

設計書には以下7セクションが含まれます:

1. **システム全体構成** — 全データの流れを1枚図で
2. **データソース一覧と取得方式** — 14源それぞれの取得スクリプト/GCS保存先
3. **データフロー** — GCS → BigQuery 3層構造への取込み経路
4. **BigQueryテーブル仕様** — 各テーブルのスキーマ・パーティション・行数
5. **業務ロジック** — フリー在庫・販売速度・推奨発注数の計算式
6. **運用情報** — 実行スケジュール・エラー対応・シークレット管理
7. **Phase 1 完了確認 + Phase 2 引き継ぎ** — 達成チェック + 未着手項目の引き継ぎ

## 📌 完了書 = Phase 1 終了の宣言と受領サイン

完了書は下記の要素で構成:

- Phase 1 達成項目チェックリスト (データ取得 / データ基盤 / 業務ロジック / 運用)
- 納品物一覧 (ドキュメント / システム / GCPリソース)
- Phase 2 への引き継ぎ事項
- 受領確認サインオフ欄
