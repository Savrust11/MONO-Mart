# ZOZO同一品番併売分析スキル

ZOZOの併売データ（CSV）から特定のブランド品番の「同一品番併売（色違い・サイズ違い）」を分析し、カラー展開の拡張提案とリピート発注優先度の評価を行う。

## 前提条件

- 分析対象となるZOZOの併売データCSVファイル（`shift_jis`エンコーディング、28カラム）がアップロードされていること
- 分析対象の「ブランド品番」が指定されていること

## ワークフロー

```
1. データ分析の実行（analyze_same_item.py）
2. 出力方法の選択
   2a. PDFレポート生成（generate_report.py）
   2b. Webアプリ表示用JSONデータ生成
   2c. Excelエクスポート（CSV形式）
3. 結果報告
```

## Step 1: データ分析の実行

```bash
python3 /home/ubuntu/skills/zozo-same-item-cross-buy/scripts/analyze_same_item.py \
  <csv_path> <brand_item_code> \
  [--compare <compare_item_code>] \
  [--brand-key <brand_key>] \
  --out <output_json_path>
```

| 引数 | 必須 | 説明 |
|---|---|---|
| `csv_path` | ○ | 併売データCSVのパス |
| `brand_item_code` | ○ | 分析対象のブランド品番（例: `pt165`） |
| `--compare` | × | 比較品番。省略時は同カテゴリ×同性別で併売率上位の品番を自動選択 |
| `--brand-key` | × | ブランドキー。省略時はCSVのショップ名から自動判定 |
| `--out` | ○ | 分析結果JSONの出力先パス |

出力JSONの構造:
- `target`: 対象品番の分析結果（併売率、カラー展開、色ペアTOP10、カテゴリペア構成比、リピートスコア、粗利率等）
- `compare`: 比較品番の分析結果（同上）
- `category`: カテゴリ全体の統計（品番数、平均併売率、ランキング、メインカラー数別平均併売率、平均リピートスコア、平均粗利率）

## Step 2a: PDFレポート生成

```bash
python3 /home/ubuntu/skills/zozo-same-item-cross-buy/scripts/generate_report.py \
  <analysis_json_path> --out <output_pdf_path>
```

### セクション折りたたみ機能

セクション1・2は右端の▼/▲ボタンでコンテンツを非表示（折りたたみ）にできる。

- **セクション1のみ非表示**: セクション2が左カラム、セクション3が右カラムの2カラムレイアウト
- **セクション2のみ非表示**: セクション1が左カラム、セクション3が右カラムの2カラムレイアウト
- **両方非表示**: セクション1・2のヘッダーバーのみ表示し、セクション3がフル幅で拡大表示

セクション3（品番評価）には折りたたみボタンなし。

### リピートスコアの吹き出し順位バッジ

リピートスコアカードの右上に吹き出し風バッジで「◎1位」のようにカテゴリ内順位を表示する。

- **TOP2以内**: ◎マーク付き（金色）で強調
- **3位以下**: 順位数字のみ
- **対象品番側**: 吹き出しをやや大きく表示（`primary`フラグ）

順位計算は `useAnalysis.ts` 内で、同一カテゴリの全品番を `repeatScore` 降順でソートして算出する。

### スコア内訳の表示

重み（%）を表示せず、各要素のスコアに「点」を付けて表示する。

```
粗利率  100点    販売数  80点    季節安定性  80点    併売率  86点
```

### カラーパレットモーダル（リピート推奨カラーの具体例）

結論ボックスの下に「リピート推奨カラーの具体例」ボタンを配置。クリックでモーダルを表示する。

#### モーダル構成

1. **ヘッダー**: 「リピート推奨カラーの具体例」タイトル＋「推奨カラー」ボタン（赤色、目立たせる）
2. **ブランド情報**: ブランド名・テイスト＋説明テキスト
3. **カラーカテゴリ別表示**: メイン・シーズン・アクセントの3カテゴリ。各色はクリックで選択/解除可能
4. **選択状況バー**: 選択数表示＋「発注Excelエクスポート」ボタン＋「発注推奨データ」ボタン（下部固定）

#### 推奨カラー画面（「推奨カラー」ボタン押下時）

結論の推奨カテゴリ（例: シーズンカラー）をクローズアップ表示:

- **保有済み色**: 緑チェックマーク＋「保有済」ラベル
- **新色候補**: 点線枠＋「NEW」ラベル。クリックで選択可能
- **提案サマリ**: 「sh1599は現在シーズンカラーが2色（モカ・コーヒーブラウン）です。パレットには他にダークネイビー・ライトグレー・ミリタリーカーキがありますが…まずはダークネイビー・ライトグレーを新色として追加することをおすすめします。」
- **カラーカテゴリ表示**: メイン・シーズン・アクセントの全色を表示（推奨カテゴリは半透明化）

#### 発注推奨数量表示（推奨カラー画面・アクセントカラーの下に常時表示）

推奨カラー画面（`showRecommendation` ON時）のカラーカテゴリ表示の下（アクセントカラーの後）に、発注推奨数量を常時表示する。色を選択しなくても品番全体推奨数が表示され、新色を選択すると追加で新色合計・1色あたり・サイズ別配分も展開される。

| 表示項目 | 条件 | 説明 |
|---|---|---|
| 品番全体推奨数 | 常時表示 | `item_scores.json` の `rec_with_penalty` 値。在庫持ち月数も表示 |
| 新色合計推奨数 | 新色選択時 | 新色の想定シェア × 品番全体推奨数（既存色の平均シェア × 0.8、上限50%） |
| 1色あたり | 新色選択時 | 新色合計推奨数 ÷ 選択色数 |
| サイズ別配分 | 新色選択時 | 品番の `size_ratio` に基づく1色あたりのサイズ別発注数 |
| ガイドメッセージ | 新色未選択時 | 「新色をクリックで選択すると、新色別の推奨数量・サイズ別配分も表示されます」 |

データソース: `useOrderRecommendation.ts` フックが CDN上の `item_scores.json` を取得し、`calcNewColorOrder()` で算出。

## リピートスコアの定義

100点満点で品番ごとのリピート発注優先度を評価する。

### スコア構成

| 要素 | 重み | 説明 |
|---|---|---|
| 粗利率スコア | **40%** | 粗利率が高いほど高スコア。収益性の最重要指標 |
| 販売数スコア | **25%** | 販売実績が多いほど高スコア |
| 季節安定性スコア | **20%** | 月別の販売変動が小さいほど高スコア |
| 併売率スコア | **15%** | 同一品番の色違い併売率が高いほど高スコア |

### 判定基準と在庫持ち月数

| スコア | 判定 | 在庫持ち月数 |
|---|---|---|
| 75〜100 | 最優先リピート | 5.0ヶ月 |
| 60〜74 | 優先リピート | 4.0ヶ月 |
| 40〜59 | 標準リピート | 3.0ヶ月 |
| 25〜39 | 慎重リピート | 2.5ヶ月 |
| 0〜24 | 最小限リピート | 2.0ヶ月 |

### ペナルティ（合計上限30pt）

| ペナルティ | 発動条件 | 計算式 |
|---|---|---|
| GM低下ペナルティ | 直近2ヶ月のGMが -2pp以上低下 | 低下幅 × 0.8 |
| 粗利率絶対値ペナルティ | 粗利率が 30%未満 | (30 - 粗利率) × 1.0 |

## カラー分類の定義

カラーを **メインカラー・シーズンカラー・アクセントカラー** の3つに分類する。

| 分類 | 説明 |
|---|---|
| メインカラー | 定番色。無彩色・モノトーン系が中心 |
| シーズンカラー | 季節感のある中間色。メインカラーとの組み合わせが多い |
| アクセントカラー | 鮮やかな色や個性的な色。上記いずれにも該当しない色 |

分類ルール詳細: `references/color_classification.md`
ブランド別カラーパレットJSON: `references/brand_color_palettes.json`
ブランド別カラーパレット（HEXコード付き）: `references/brand_color_palettes_hex.json`

## 結論ロジック

分析結果に基づき、カラー展開の推奨を自動生成する。

| 条件 | 推奨 |
|---|---|
| メインカラー同士の併売ペアが15%以上 & メインカラー追加で併売率向上が見込める | **メインカラー追加を最優先で推奨** |
| シーズンカラーが1色以下 & シーズン×メインの併売が15%未満 | **シーズンカラー追加を推奨** |
| 上記以外 | **メインカラー追加を推奨**（汎用性が高いため） |

### 推奨カラー提案ロジック

結論の推奨カテゴリに基づき、カラーパレットから具体的な新色を提案する:

1. 推奨カテゴリ（メイン/シーズン/アクセント）のパレット色を取得
2. 品番が既に保有している色名と照合し「保有済み」「新色候補」に分類
3. 新色候補の先頭2色を優先推奨として提案サマリに記載
4. ユーザーがクリックで選択した色を発注Excelに反映

## ユーザーへの報告

1. 分析サマリー（対象品番の併売率、リピートスコア・カテゴリ内順位、色ペアの傾向、結論の要約）

3. 使用したブランドキーとカラー分類の説明
4. 自動選択された比較品番についての説明（ユーザー指定がない場合）

注文データが更新された場合:
1. `aggregate_orders.py` を再実行（Google DriveのCSVを順次処理）
2. `calc_scores_and_orders.py` を再実行
3. 生成された `item_scores.json` を `manus-upload-file --webdev` でCDNにアップロード
4. `useOrderRecommendation.ts` のURL定数を新しいCDN URLに更新
