# ZOZO 在庫消化予測スキル

ZOZOの52週カテゴリ別売上係数を用いて、品番単位の在庫消化予測を行う。

## ワークフロー

在庫消化予測は以下の手順で実行する:

1. **52週係数の生成**（`scripts/build_coefficients.py`を実行）
2. **在庫消化予測の実行**（`scripts/forecast_inventory.py`を実行）
3. **結果の分析と報告**（CSVを読み込みレポート作成）

必要に応じて `scripts/show_coefficients.py` で特定カテゴリの係数を確認できる。

## Step 1: 52週係数の生成

【52週】カテゴリ推移.xlsx からカテゴリ(子)別の52週係数JSONを生成する。

```bash
python scripts/build_coefficients.py <52週Excel> <出力JSON>
```

- 入力: ZOZOの52週カテゴリ推移Excel（「ZOZO全体」データを含むシート）
- 出力: カテゴリ(子)ごとの週別構成比JSON
- 「ZOZO全体」行のみ集計（「社内」行は除外）
- 処理に数分かかる場合がある（35万行超）

**既に係数JSONが手元にある場合はこのステップをスキップ可能。**

## Step 2: 在庫消化予測

品番別在庫Excelと係数JSONから、指定期日までの消化予測を実行する。

```bash
python scripts/forecast_inventory.py <在庫Excel> <係数JSON> <期限YYYY-MM-DD> [オプション]
```

主要オプション:

| オプション | 説明 | デフォルト |
|-----------|------|-----------|
| `--sheet NAME` | 対象シート名 | 最初のシート |
| `--targets A,B,C` | 対象品番（カンマ区切り） | 全品番 |
| `--recent-days N` | 日販計算の直近日数 | 14 |
| `--output PATH` | 出力CSVパス | forecast_results.csv |

### 計算ロジック

1. 各品番の直近N日の平均日販を算出
2. カテゴリ(子)の52週係数で週ごとに日販を補正: `予測日販 = 平均日販 × (対象週係数 / 現在週係数)`
3. 予測期限まで日ごとに積み上げて累計予測販売数を算出
4. `消化率 = 予測販売数 / 現在庫 × 100`

### 日販の算出

- 日別販売データの末尾連続0を除外して実データ期間を自動検出
- 直近N日（デフォルト14日）の平均を使用
- 直近が0の場合は全期間平均にフォールバック

### 週番号の定義

`week = min((day_of_year - 1) // 7 + 1, 52)` — 1/1～1/7が第1週。

## Step 3: 結果の分析と報告

出力CSVの列: ブランド, 品番, 商品名, カテゴリ(子), シーズン, 現在庫, 直近日販, 予測販売数, 残在庫, 消化率(%), 判定

報告時のポイント:

- **余剰品番**（残在庫 > 0）を残在庫の多い順に提示
- 消化率帯別に分類: 0-10%（不動）、10-30%、30-50%、50-70%、70-100%、100%超（完売見込）
- カテゴリ別の季節特性に言及（例: パーカーは春夏に係数急落、シャツは春夏に係数上昇）

## 係数の確認（ユーティリティ）

特定カテゴリの週別係数を確認する:

```bash
python scripts/show_coefficients.py <係数JSON> <カテゴリ名> [--weeks 13-22] [--base-week 13]
```

## データフォーマット

Excel列構成の詳細は `references/data-format.md` を参照。

### 在庫Excel（MONO-MART形式）の必須列

- B列: ブランド品番（キー）
- G列: カテゴリ(子)（係数との紐付け）
- M列: 在庫数
- O列以降: 日別販売数（ヘッダーにdatetime型日付）

## 注意事項

- 52週Excelの読み込みは35万行超のため数分かかる
- 係数JSONは年度ごとに1回生成すれば再利用可能
- 夏物カテゴリは3月時点で日販が低いが、係数で5月以降の上昇も反映済み
- 在庫0の品番でも日販がある場合は予測販売数が出る（入荷前品番等）
