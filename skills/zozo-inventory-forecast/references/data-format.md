# データフォーマット仕様

## 52週カテゴリ推移Excel（ZOZO全体データ）

ファイル名例: `【52週】カテゴリ推移.xlsx`

### ZOZO全体シート

全ショップ・全カテゴリの商品別販売実績（年間）。

| 列 | 内容 |
|----|------|
| カテゴリ(子) | 子カテゴリ名（パーカー、Tシャツ/カットソー等） |
| 週 | 1～52の週番号（1月1週=1、12月最終週=52） |
| 販売数 | 当該週の販売数量 |

週番号の定義: `week = min((day_of_year - 1) // 7 + 1, 52)`

### 係数JSON出力形式

```json
{
  "パーカー": {
    "total_sales": 123456,
    "weekly_coefficients": {
      "1": 0.01234,
      "2": 0.00987,
      ...
      "52": 0.01100
    }
  }
}
```

各週の係数 = 当該カテゴリの当該週販売数 / 年間合計販売数

## 品番別在庫Excel（MONO-MART形式）

ファイル名例: `MONO-MART_2026_3.xlsx`

### ヘッダー行（Row 3）

| 列 | フィールド | 説明 |
|----|-----------|------|
| A | ブランド | ブランド名 |
| B | ブランド品番 | 品番コード（ユニークキー） |
| C | 商品名 | 商品名称 |
| D | 元上代(税抜き) | 税抜き上代 |
| E | 下代単価 | 仕入単価 |
| F | カテゴリ(親) | 親カテゴリ（トップス、パンツ等） |
| G | カテゴリ(子) | 子カテゴリ（パーカー、スラックス等）※係数との紐付けキー |
| H | シーズン | 春/秋、夏、通年、秋/冬 等 |
| I | 種類 | 任意 |
| J | テイスト | 任意 |
| K | 26SS入荷残 | SS入荷予定残数 |
| L | 26AW入荷残 | AW入荷予定残数 |
| M | 在庫数 | 現在庫数 |
| N | 売れた合計 | 期間内の販売合計 |
| O～ | 日別販売数 | ヘッダーにdatetime型の日付が入る |

### 注意事項

- データ行は Row 4 から開始
- 日付列はヘッダーがdatetime型で自動検出可能
- 未来日の列は0埋めされている（データ最終日の判定に使用）
- Row 1-2はサマリ行（スキップ対象）
