## 1. 発注表シート

メインシート。SKUデータ・発注数・10日刻み売上実績・レポート画像・商品画像を含む。

### ヘッダー部（Row 1〜13）

| セル | 内容 |
|------|------|
| E12 | 品番（ブランド品番を入力） |
| Row 7 | 予約の10日刻み合計行 |
| Row 8 | 実売の10日刻み合計行 |
| Row 11 | 10日刻みヘッダー行（年月） |
| Row 13 | 列ヘッダー行 |

### SKUデータ部（Row 14〜）

各SKU（カラー×サイズ）1行ずつ。

| 列 | 列名 | データソース |
|----|------|-------------|
| A | 連番 | 自動採番（1, 2, 3...） |
| B | メーカー品番 | 品番（全行同じ） |
| C | メーカーカラー名 | 在庫分析CSV: カラー |
| D | （空き） | - |
| E | 品番 | ブランド品番 |
| F | カラー | 在庫分析CSV: カラー |
| G | サイズ | 在庫分析CSV: サイズ |
| H | CS品番 | 在庫分析CSV: CS品番 |
| I〜K | （数式参照列） | フォーマットの既存数式 |
| L | 在庫 | 在庫分析CSV: 販売可能数 |
| M〜N | （数式参照列） | フォーマットの既存数式 |
| O | 販売数（30日） | 在庫分析CSV: 直近30日販売数 |
| P | （数式参照列） | フォーマットの既存数式 |
| Q | お気に入り | 在庫分析CSV: お気に入り登録数 |
| R | SKU | 品番 + CS品番（例: sc841123456789） |
| S | 7日間 | 在庫分析CSV: 直近7日販売数 |
| T | 一ヶ月 | 在庫分析CSV: 直近30日販売数 |
| **U** | **発注数** | **SKUバランスV2配分結果** |
| V | フリー在庫 | 在庫 - 予約未処理数 |
| W | 7日間分 | フリー在庫 / (7日販売/7)（在庫日数） |
| X | 1ヶ月差分 | フリー在庫 / (30日販売/30)（在庫日数） |
| Y〜Z | （空き） | - |
| AA | 予約未処理数 | 予約管理一覧CSV: 未処理 |
| AB | お気に入り（直近1ヶ月） | 在庫分析CSV: お気に入り登録数 |
| AC〜AF | 年度別平均売価 | AC:2023年, AD:2024年, AE:2025年, AF:2026年 |

### 10日刻み売上実績（AG列〜）

AG列（列番号33）から始まり、3列で1ヶ月分（1~10日, 11~20日, 21~31日）。

**列番号の計算式:**
```
col = 33 + month_offset * 3 + ten_day_offset
```

| 年 | month_offset |
|----|-------------|
| 2023年1月 | 0 |
| 2023年12月 | 11 |
| 2024年1月 | 12 |
| 2024年12月 | 23 |
| 2025年1月 | 24 |
| 2025年12月 | 35 |
| 2026年1月 | 36 |
| 2026年6月 | 41 |

ten_day_offset: `1~10` → 0, `11~20` → 1, `21~31` → 2

**最大列:** FB列（列番号158）まで

**書き込み対象:**
- Row 7: 予約販売の10日刻み合計
- Row 8: 実売（通常販売）の10日刻み合計
- Row 14〜: 各SKUの10日刻み販売数

### レポート画像エリア

SKUデータの最終行の3行下から配置。satori/sharpで生成したPNG画像をS3にアップロードし、IMAGE関数で埋め込む。

```
report_start_row = last_data_row + 3
```

- B〜Z列を結合してIMAGE関数を配置
- 行高さ: 画像の高さに応じて自動調整

### 商品画像エリア

レポート画像の下（レポート画像がない場合はSKUデータの最終行の3行下）から配置。

```
img_start_row = report_end_row + 2  // カラー名行
img_row = img_start_row + 1          // IMAGE関数行
```

- 各カラー1枚ずつ横並び（**1列間隔: B,C,D,E,F,G,H,...列**）
- 画像サイズ: 幅120px × 高さ150px
- 画像の1行上にカラー名を記入

**Google Sheets版（IMAGE関数方式）:**

| 項目 | 値 |
|------|------|
| 関数 | `=IMAGE("url", 4, 150, 120)` |
| モード | 4（カスタムサイズ） |
| 高さ | 150px |
| 幅 | 120px |
| 画像列幅 | **135px**に自動調整（IMAGE関数の幅120pxが収まるように） |
| 画像行高さ | **165px**に自動調整 |
| URLパターン | `https://o.imgz.jp/{末尾3桁}/{商品コード}/{商品コード}b_{カラーコード}_d.jpg` |

**注意:**
- 初回表示時にGoogle Sheetsの「Allow access」クリックが必要
- 画像列の幅が120px未満だと画像が表示されない
- テンプレートに埋め込み画像を含めないこと（コピー時に引き継がれる）
- 旧仕様では2列間隔（B,D,F,...）で配置していたが、カラー数が多い品番（8色以上）で画像が途切れる問題があったため1列間隔に変更

## 2. 売上実績シート

注文生データを書き込むシート。

### 列構成（A〜AB列）

**データ列（A〜S列）:**

| 列 | 列名 | データソース |
|----|------|-------------|
| A | ショップ名 | 注文生データ |
| B | 親カテゴリ | 注文生データ |
| C | 子カテゴリ | 注文生データ |
| D | 親商品タイプ | 注文生データ |
| E | 子商品タイプ | 注文生データ |
| F | 性別 | 注文生データ |
| G | ブランド品番 | 注文生データ |
| H | 商品名 | 注文生データ |
| I | CS品番 | 注文生データ |
| J | カラー | 注文生データ |
| K | サイズ | 注文生データ |
| L | 販売価格（税抜） | 注文生データ（数値変換） |
| M | 販売タイプ | 注文生データ |
| N | 価格タイプ | 注文生データ |
| O | プロパー価格（税抜） | 注文生データ（数値変換） |
| P | 注文数 | 注文生データ（数値変換） |
| Q | 合計金額（税抜） | 注文生データ（数値変換） |
| R | 注文日 | 注文生データ |
| S | バーコード | 注文生データ |

**計算列（T〜AB列）:**

| 列 | 列名 | 算出方法 |
|----|------|----------|
| T | SKU | ブランド品番 + CS品番 |
| U | 年 | 注文日から抽出 |
| V | 月 | 注文日から抽出 |
| W | 日 | 注文日から抽出 |
| X | 前後半 | 日≤15→「前半」、日>15→「後半」 |
| Y | 10日刻み | 日≤10→「1~10」、日≤20→「11~20」、日>20→「21~31」 |
| Z | 5日ごと | 5日区切り（例:「1~5」「6~10」...） |
| AA | 3日ごと | 3日区切り（例:「1~3」「4~6」...） |
| AB | 日付変換 | datetime型の日付 |
