# MD計画 データソース定義

## 1. 買い回りデータ（外部環境分析用）

- **形式**: Excel (.xlsx)、ブランドごとにシート分割
- **Google Drive**: 手動ダウンロード（ファイル名例: `買い回りデータ2026年1月.xlsx`）
- **列定義（18列）**: 会員性別, 年齢区分, ショップID, ショップ名, 親カテゴリ, 子カテゴリ, 商品親タイプ, 商品子タイプ, 商品コード, ブランド品番, 商品名, 商品主性別, 上代, 販売価格, 販売タイプ, 価格タイプ, 売上点数, 売上金額
- **カテゴリ列**: `商品親タイプ`（トップス、パンツ等）。`子カテゴリ`はブランド名が入っているため使用しない。
- **シート名 = ブランド名**: 各シートに記載されているデータは、そのブランド購入者が同月内に他社で購入した商品。

## 2. ショップ指標データ（外部環境分析用）

- **形式**: Excel (.xls)、Sheet1のみ使用
- **Google Drive**: フォルダ名 `ZOZO集計データ` 等で検索
- **読み込み注意**: ヘッダーが2行目にある場合あり → `header=None`で読み込み、行0をカラム名として使用
- **必要列**: ショップ名、購入者数（相性スコアの母数）

## 3. 注文データ（内部環境分析用）

- **詳細**: `zozo-order-data` スキル参照
- **列ずれ注意**: ヘッダー28列に対しデータ行29フィールド（先頭に親ショップ名「MONO-MART」が追加）
- **正しい列マッピング**: `scripts/md_plan_step2_internal.py` の `CORRECT_COLUMNS` 定数を参照

## 4. 原価データ（内部環境分析用）

- **形式**: CSV
- **Google Drive**: `sales_data/cost_data.csv`
- **列定義**: ブランド品番, 商品名, 販売価格, 原価, 粗利率
- **用途**: 品番レベルの粗利率を注文データに結合
